Mini Project:  Food Delivery Console App
Objective:
ASSESSMENT FILE
Java
M2-A1
Build a console-based food ordering application that lets a customer browse the menu, place
orders, and view order history, combining OOP with encapsulation, ArrayList-based cart
management, JDBC PreparedStatement queries, and atomic transaction control into one
working program.
Your project must:
The program must be menu-driven with at least four numbered options: (1) Browse Menu, (2)
Place Order, (3) View My Order History, (4) Exit — and must loop until the customer selects Exit.
At startup, load the menu from a menu_items table in MySQL using a JDBC
PreparedStatement SELECT; store the results in an ArrayList<MenuItem> and display them with
item ID, name, price, and availability status.
When the customer places an order, execute the wallet balance deduction and the order
insertion as a single JDBC transaction (auto-commit off); if either step fails — for example due
to insufficient balance — roll back both changes and display a clear error message without
crashing the program.
Order history must be fetched from the database using a PreparedStatement parameterised
by the customer ID and displayed as a formatted table showing order ID, restaurant name,
amount, and date.
All JDBC resources must be managed with try-with-resources; non-numeric input, negative
amounts, and empty table results must all be handled with appropriate messages rather than
unhandled exceptions.
